## Catatan Perapian Folder

- File pada folder ini dipertahankan hanya untuk kebutuhan pembelajaran, operasional, dan referensi implementasi.
- Penomoran file dipakai agar urutan pembelajaran konsisten untuk pengguna baru.
- Aset HTML duplikat yang tidak menjadi jalur baca utama sengaja dibersihkan dari folder ini agar pengguna tidak bingung memilih dokumen.

## Catatan Keamanan Dokumentasi

- Jangan menaruh password, API key, token, atau kredensial production secara hardcode di file Markdown.
- Untuk contoh konfigurasi, gunakan placeholder seperti `<SATRIA_HOST>`, `<IRIS_URL>`, `<API_TOKEN>`, atau `<JENKINS_SERVER_HOST>`.
- Bila dokumen perlu mengarah ke sistem production, gunakan tautan generik, nama menu, atau placeholder yang aman.
